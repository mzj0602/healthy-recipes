# Pipeline 模型配置

各阶段使用的 Claude 模型。**修改此文件即可全局调整模型分配**，P0 调度器会读取此配置。

| 阶段 | 模型 | 原因 |
|------|------|------|
| P1 需求分析 | haiku | 结构化文档输出，规则明确，不需要复杂推理 |
| P2 技术设计 | opus | 架构决策与权衡，需要最强推理能力 |
| P2b UI 设计 | sonnet | 从 Stitch 获取或生成界面设计稿，供开发参考 |
| P3 任务拆解 | sonnet | 中等复杂度，结合设计转化为可执行任务 |
| P4 开发 | sonnet | 编码主力，质量与速度均衡 |
| P5 代码 Review | sonnet | 协调 Codex 多轮验证，修复问题；Codex 负责实际审查 |
| P6 测试 | sonnet | 测试执行、结果分析与验证 |
| P7 文档同步 | haiku | 简单文档更新，规则明确 |
| P8 提交 | haiku | Git 操作，无需复杂推理 |

## 可用模型

- `opus` — claude-opus-4-6（最强，适合推理/设计/审查）
- `sonnet` — claude-sonnet-4-6（均衡，适合编码/测试）
- `haiku` — claude-haiku-4-5（最快，适合简单结构化任务）
