# header-search 技术设计

**版本**：2026-03-23
**关联需求**：specs/header-search/2026-03-23-requirements.md

## 方案概述

在 `SiteChrome` 中新增 `HeaderSearch` 子组件，实现搜索图标展开/收起动画与关键词提交；将 `RecipeExplorePage` 的本地 `query` state 迁移至全局 `searchAtom`，使 Header 与探索页共享同一搜索状态；提交时通过已有的 `onNavigate` prop 切换至 `recipes` 页面，无需新增任何 tRPC procedure。

## 影响范围

- **前端**：
  - `src/features/meal-planner/components/site-chrome.tsx`（修改，新增 HeaderSearch 内联组件）
  - `src/features/meal-planner/components/recipe-explore-page.tsx`（修改，迁移 query state → searchAtom）
- **Worker**：无变更
- **共享类型**：无变更

## 数据模型

无新类型。复用已有：

```typescript
// src/features/recipes/store/recipeAtoms.ts（现有，无需修改）
export const searchAtom = atom('');         // 全局搜索关键词
export const filteredRecipesAtom = atom(…); // 已依赖 searchAtom
```

## 前端方案

### 新增组件

| 组件名 | 路径 | 职责 |
|--------|------|------|
| `HeaderSearch`（内联） | `site-chrome.tsx` 内 | 管理展开/收起 UI 状态，写入 searchAtom，调用 onNavigate |

> 因为 `HeaderSearch` 只被 `SiteChrome` 使用，不单独提取文件，内联在 `site-chrome.tsx` 中。

### 状态设计

| 状态 | 类型 | 位置 | 说明 |
|------|------|------|------|
| `isExpanded` | `boolean` | `HeaderSearch` 本地 `useState` | 控制搜索框展开/收起，仅影响当前组件 UI |
| `localValue` | `string` | `HeaderSearch` 本地 `useState` | 输入中间态，按 Enter / 点击搜索才写入全局 atom |
| `searchAtom` | `string` | Jotai（已有） | 全局搜索关键词，`RecipeExplorePage` 读取 |

**为什么用本地 `localValue` 而非直接写 `searchAtom`？**
直接写会导致每次键入就触发 `filteredRecipesAtom` 重算，产生实时过滤效果，与需求（按 Enter 才跳转）不符，且会在其他页面可见地改变菜谱列表状态。

### 路由/导航

```
用户在 HeaderSearch 按 Enter
  → localValue 非空 → useSetAtom(searchAtom)(localValue)
                     → onNavigate('recipes')
  → localValue 为空 → 不跳转，不触发

若 activePage 已为 'recipes'
  → 只写 searchAtom，不再调用 onNavigate（避免页面重新挂载）
```

`onNavigate` 已经通过 `SiteChromeProps` 传入，`HeaderSearch` 以 prop 形式接收 `activePage` 和 `onNavigate`。

### RecipeExplorePage 迁移

**改动最小化**：仅将

```typescript
const [query, setQuery] = useState('');
```

替换为

```typescript
const [query, setQuery] = useAtom(searchAtom);
```

其余过滤逻辑（`useMemo` + `filteredRecipes`）保持不变，因为 `searchAtom` 与原 `query` 类型相同（`string`）。

### HeaderSearch 组件设计

```tsx
// 在 site-chrome.tsx 内，SiteChrome 函数之前定义
interface HeaderSearchProps {
  activePage: PlannerPageId;
  onNavigate: (page: PlannerPageId) => void;
}

function HeaderSearch({ activePage, onNavigate }: HeaderSearchProps) {
  const [isExpanded, setIsExpanded] = useState(false);
  const [localValue, setLocalValue] = useState('');
  const setSearch = useSetAtom(searchAtom);
  const inputRef = useRef<HTMLInputElement>(null);

  // 展开后自动聚焦
  useEffect(() => {
    if (isExpanded) inputRef.current?.focus();
  }, [isExpanded]);

  // 点击外部收起：通过 onBlur + 延迟（避免 click 竞争）
  function handleBlur() {
    setTimeout(() => setIsExpanded(false), 150);
  }

  function handleSubmit() {
    const trimmed = localValue.trim();
    if (!trimmed) return;
    setSearch(trimmed);
    if (activePage !== 'recipes') onNavigate('recipes');
    setIsExpanded(false);
    setLocalValue('');
  }

  function handleKeyDown(e: React.KeyboardEvent) {
    if (e.key === 'Enter') handleSubmit();
    if (e.key === 'Escape') { setIsExpanded(false); setLocalValue(''); }
  }

  return (
    <div className="relative flex items-center">
      {/* 展开时渲染 input，用 CSS transition 控制宽度 */}
      <div className={cn(
        'overflow-hidden transition-all duration-200',
        isExpanded ? 'w-[220px] opacity-100' : 'w-0 opacity-0'
      )}>
        <input ref={inputRef} value={localValue} onChange={…} onBlur={handleBlur} onKeyDown={handleKeyDown} … />
      </div>
      <button onClick={() => setIsExpanded(true)} …>
        {/* 放大镜 SVG */}
      </button>
    </div>
  );
}
```

**收起触发方式**：使用 `onBlur + setTimeout(150ms)` 而非 `document.addEventListener('click')`，避免跨层事件监听，也不需要额外的 `useEffect` 清理。

## API 方案

无，本次为纯前端状态操作。

## 安全考量

- 无 API 调用，无 XSS 风险（React 自动转义输入）
- `localValue` 传给 `searchAtom` 前无需额外过滤（仅用于前端字符串匹配）

## 测试策略

### 单元测试

文件：`src/test/unit/header-search.test.tsx`

| 测试用例 | 验证点 |
|---------|--------|
| 点击搜索图标后 input 出现 | `isExpanded` 变为 true |
| 输入空字符串按 Enter | `searchAtom` 不被写入，`onNavigate` 不被调用 |
| 输入关键词按 Enter | `searchAtom` = 关键词，`onNavigate('recipes')` 被调用 |
| 已在 recipes 页搜索 | `onNavigate` 不被重复调用 |
| 按 Esc 键 | input 收起，`localValue` 清空 |

### E2E 测试

文件：`src/test/e2e/header-search.spec.ts`

```
流程 1：从首页通过 Header 搜索跳转至探索页

  - 输入"三文鱼"
  - 按 Enter
  - 验证：探索页搜索栏显示"三文鱼"，菜谱列表已过滤

流程 2：已在探索页，Header 搜索直接更新列表
  - 导航至菜谱探索页（点击导航"菜谱"）
  - 点击 header 搜索图标，输入"高蛋白"，按 Enter
  - 验证：菜谱列表刷新，无页面切换闪烁

流程 3：空搜索不跳转
  - 点击 header 搜索图标
  - 不输入，直接按 Enter
  - 验证：页面仍为首页

流程 4：点击外部收起
  - 展开搜索框
  - 点击搜索框外部区域
  - 验证：搜索框收起
```

注意：E2E 测试不使用 `page.goto()`，全程通过点击 UI 导航。

## 不在设计范围内

- 搜索联想/自动补全
- 搜索历史记录
- 移动端响应式
- 非菜谱内容搜索
- `RecipeExplorePage` 现有侧边栏筛选逻辑（热量、过敏原等）迁移至 Jotai atom
