## 前端方案

### 新增组件

| 组件 | 路径 | 职责 |
|---|---|---|
| `EditPlanModal` | `src/features/health-plan/components/edit-plan-modal.tsx` | 弹窗主体：草稿状态管理、Tab 切换、营养目标编辑、餐食增删改、校验、保存 |

### 状态设计

弹窗内状态全部用 `useState`（局部），无需 Jotai atom（编辑草稿不需要跨组件共享）：

| State | 类型 | 说明 |
|---|---|---|
| `draftPlan` | `DayPlan[]` | 深拷贝自 `plan` props，编辑中间态 |
| `draftGoals` | `NutrientGoal[]` | 深拷贝自 `goals` props，编辑中间态 |
| `activeDayId` | `string` | 当前选中的日期 Tab，默认 `plan[0].id` |
| `errors` | `Record<string, string>` | 校验错误，key 格式：`${mealId}-title` / `${mealId}-calories` |

`HealthPlanPage` 侧新增：

| State | 类型 | 说明 |
|---|---|---|
| `plan` | `DayPlan[]` | 当前生效计划，初始值从 `localStorage` 读取 |
| `goals` | `NutrientGoal[]` | 当前生效营养目标，初始值从 `localStorage` 读取 |
| `isEditOpen` | `boolean` | 弹窗开关 |

### 持久化逻辑（HealthPlanPage）

```typescript
// 读取（带容错）
function loadFromStorage(): { plan: DayPlan[]; goals: NutrientGoal[] } {
  try {
    const raw = localStorage.getItem('health-plan-data');
    if (!raw) throw new Error('empty');
    const parsed = JSON.parse(raw) as HealthPlanStorage;
    if (!Array.isArray(parsed.plan) || !Array.isArray(parsed.goals)) throw new Error('invalid');
    if (parsed.plan.some((d) => !Array.isArray(d.meals))) throw new Error('invalid');
    return parsed;
  } catch {
    return { plan: defaultPlan, goals: defaultGoals }; // 回退默认数据
  }
}

// 写入（带容错，Safari 隐私模式兼容）
function handleSave(newPlan, newGoals) {
  try { localStorage.setItem('health-plan-data', JSON.stringify({ plan: newPlan, goals: newGoals })) }
  catch { /* 存储不可用，仅内存更新 */ }
  setPlan(newPlan); setGoals(newGoals); setIsEditOpen(false);
}
```

### 路由 / 导航

SPA 状态路由，无 URL 变化。弹窗通过 `isEditOpen` 状态控制挂载/卸载：`{isEditOpen && <EditPlanModal ... />}`

## 表单校验方案

点击"保存计划"时遍历全部七天所有餐食（非仅当前 Tab）：

| 条件 | 错误 key | 错误文案 |
|---|---|---|
| `meal.title.trim() === ''` | `${mealId}-title` | 餐食名称不能为空 |
| `meal.calories < 0 \|\| !Number.isFinite \|\| !Number.isInteger` | `${mealId}-calories` | 请输入有效卡路里（≥ 0） |

有错误时：不关闭弹窗、不写 localStorage、不调用 `onSave`。

`percent` 字段在 `onChange` 时实时截断：`Math.round(Math.min(100, Math.max(0, Number(value) || 0)))`，无需保存校验。

## API 方案

**不涉及 Worker**。本次完全为纯前端 localStorage 持久化，无 tRPC procedure，无 AI 调用。

---

## 安全考量

- 仅操作 `localStorage`，无网络请求，无 XSS 风险（React 默认转义用户输入）
- `localStorage` 读取需 try/catch，防止 Safari 隐私模式下抛 `SecurityError`
- 不涉及任何 API Key，无 CORS 变更需求
