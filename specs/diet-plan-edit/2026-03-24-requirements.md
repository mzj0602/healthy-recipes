# diet-plan-edit 需求文档

**版本**：2026-03-24
**状态**：Draft

---

## 背景

FreshPlate 健康菜谱平台已上线"个人健康饮食计划"页面（`/health-plan`），展示一周七天的餐食安排和营养目标进度。页面顶部存在"编辑计划"按钮，但目前该按钮点击后的弹窗编辑功能尚未完整实现或未与持久化逻辑打通，导致用户无法自定义自己的饮食安排。需要实现完整的编辑弹窗交互，使用户能够实际修改并保存饮食计划。

---

## 目标

1. 让用户能够在弹窗中自由编辑一周七天的餐食（增删改）及营养目标，保存后立即反映在看板上。
2. 通过 `localStorage` 持久化用户编辑，刷新页面后数据不丢失。
3. 确保表单有完善的输入校验，防止无效数据写入。

---

## 用户故事

- 作为健康饮食用户，我希望点击"编辑计划"按钮后弹出编辑弹窗，以便修改每天的餐食安排。
- 作为健康饮食用户，我希望通过 Tab 切换每天的餐食，以便分日管理我的饮食内容。
- 作为健康饮食用户，我希望修改营养目标的数值和进度百分比，以便反映我实际的营养完成情况。
- 作为健康饮食用户，我希望保存后弹窗关闭且看板立即更新，以便确认我的修改已生效。
- 作为健康饮食用户，我希望取消或关闭弹窗时不保存任何草稿修改，以便我可以放心预览而不担心误操作。

---

## 功能需求

### FR-1 打开弹窗
- 点击页面顶部"编辑计划"按钮，弹出 `EditPlanModal` 对话框（`role="dialog"`）。
- 弹窗标题为"编辑计划"，包含 × 关闭按钮（`aria-label="关闭"`）和背景蒙层。

### FR-2 营养目标编辑
- 弹窗顶部"今日营养目标"区展示所有 `NutrientGoal` 条目（蛋白质、热量、碳水等）。
- 每条目提供：
  - 文本输入框：编辑 `valueLabel`（如 `(120g/150g)`）。
  - 数字输入框：编辑完成百分比 `percent`（0–100），超出范围自动截断到边界值（<0→0，>100→100）。

### FR-3 每日餐食编辑（Tab 切换）
- 弹窗下方"每日饮食计划"区展示七个日期 Tab（周一～周日）。
- 点击 Tab 切换当天餐食列表，默认显示周一。
- 每条餐食显示：
  - 餐食类型标签（`meal.type`，只读展示）。
  - 餐食名称文本输入框（`placeholder="餐食名称"`，`maxLength=100`）。
  - 卡路里数字输入框（单位 kcal，`min=0`）。
  - 删除按钮（`aria-label="删除餐食"`）。

### FR-4 添加餐食
- 每天餐食列表底部有"+ 添加餐食"按钮。
- 点击后在当前 Tab 末尾追加一条新餐食，默认 `type='其他'`，`title=''`，`calories=0`。

### FR-5 删除餐食
- 点击餐食行右侧删除按钮，从当前 Tab 移除该餐食。
- 当某天仅剩 1 条餐食时，删除按钮置为 `disabled`，防止清空当天所有数据。

### FR-6 表单校验
- 点击"保存计划"时，对**全部七天**的所有餐食执行校验：
  - `title` 为空 → 报错"餐食名称不能为空"（显示在对应输入框下方）。
  - `calories` < 0 或非有效整数 → 报错"请输入有效卡路里（≥ 0）"。
- 存在任意错误时，弹窗不关闭，不写入 `localStorage`，不调用 `onSave`。

### FR-7 保存与持久化
- 校验通过后，将 `{ plan: DayPlan[], goals: NutrientGoal[] }` 序列化写入 `localStorage`（key: `health-plan-data`）。
- 调用 `onSave(newPlan, newGoals)`，父组件更新状态，弹窗关闭。
- 页面刷新后，从 `localStorage` 恢复数据；若数据缺失或格式不合法，回退到内置默认数据。

### FR-8 取消 / 关闭弹窗
- 点击"取消"按钮、× 按钮或背景蒙层，均触发 `onClose`，弹窗关闭。
- 关闭时所有草稿状态丢弃，不写入任何数据，原计划数据保持不变。

## 验收标准

| ID | 场景 | 标准 |
|----|------|------|
| AC-1 | 点击"编辑计划"按钮 | 弹窗（role=dialog）可见，标题为"编辑计划" |
| AC-2 | 切换日期 Tab | 对应日期餐食显示，其他日期餐食隐藏 |
| AC-3 | 修改餐食名称并保存 | 弹窗关闭，看板显示新名称，刷新后仍存在 |
| AC-4 | 修改营养目标 percent 并保存 | 对应进度条百分比更新 |
| AC-5 | 取消/蒙层/× 关闭弹窗 | 弹窗关闭，看板数据不变，`onSave` 未被调用 |
| AC-6 | 添加餐食并保存 | 新行出现在看板，刷新后仍存在，新行 type 为"其他" |
| AC-7 | 删除餐食 | 对应行从列表消失；仅剩 1 条时删除按钮 disabled |
| AC-8 | 保存时 title 为空 | 错误提示显示，弹窗不关闭，localStorage 未更新 |
| AC-9 | 保存时 calories 为负 | 错误提示显示，弹窗不关闭，localStorage 未写入负值 |
| AC-10 | percent 输入超范围 | 自动截断：<0→0，>100→100 |

## 非功能需求

- **性能**：弹窗打开/关闭动画流畅（无 jank），localStorage 读写同步完成，不影响渲染帧。
- **兼容性**：支持 Chrome / Firefox / Safari 最新两个主版本；移动端 viewport 下弹窗高度不超出屏幕（`max-h-[90vh]` + 内容区可滚动）。
- **无障碍**：弹窗具有 `role="dialog"` 和 `aria-modal="true"`；关闭按钮、删除按钮有明确 `aria-label`。
- **安全**：仅写入本地 `localStorage`，无网络请求，无 XSS 风险（React 默认转义）。

---

## 不在范围内

- 餐食类型（`meal.type`）的编辑（当前为只读展示标签）。
- 增加或删除整天的计划（七天固定）。
- 营养目标条目的增删（条目固定）。
- 数据同步到云端 / 后端 API 持久化。
- "生成购物清单"按钮功能实现。

## 依赖与风险

| 项目 | 说明 |
|------|------|
| 依赖：shadcn/ui `Button` 组件 | 弹窗内"+ 添加餐食"使用 `variant="outline"` |
| 依赖：`localStorage` API | Safari 隐私模式下可能抛异常，需 try/catch 容错 |
| 风险：并发标签页数据覆盖 | 多 tab 同时编辑会导致最后保存的覆盖先保存的，当前范围内不处理 |
| 风险：E2E 路由依赖 | E2E 测试通过首页按钮导航至计划页，需确保首页入口按钮存在 |
