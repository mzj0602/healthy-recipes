# user-login 技术设计

**版本**：2026-03-29
**关联需求**：specs/user-login/2026-03-29-requirements.md

## 方案概述
纯前端轻量鉴权方案：在 `HealthyMealPlannerApp` 入口增加登录态守卫，未登录时渲染独立 `LoginPage`，登录成功后进入现有 FreshPlate SPA。前端预设演示账号，登录态通过 `sessionStorage` 保存最小必要信息（用户名），满足当前浏览器会话内保持登录的需求；退出时清除会话并回到登录页。整个方案不引入 Worker / tRPC / 后端改造，以最小改动接入现有页面结构。

## 影响范围
- 前端：
  - `src/features/meal-planner/components/healthy-meal-planner-app.tsx` — 登录态守卫、sessionStorage 读写
  - `src/features/meal-planner/components/site-chrome.tsx` — 新增 `currentUser` / `onLogout` props
  - `src/features/meal-planner/components/login-page.tsx` — 新增登录页组件
- Worker：无
- 共享类型：无新增，`LoginPayload` 定义在 `login-page.tsx` 内部

## 数据模型
```ts
interface LoginPayload {
  username: string;
}
```

浏览器会话存储 key：`healthy-recipes-user`（存用户名字符串，不存密码）

## 前端方案

### 新增组件
- `src/features/meal-planner/components/login-page.tsx`
  - Props: `onLogin: (payload: LoginPayload) => void`
  - 内部状态：`username`, `password`, `error`, `isSubmitting`
  - 演示账号：`demo / 123456`（硬编码，MVPaccepted）

### 状态设计
- 不新增 Jotai atom，登录态局限在 `HealthyMealPlannerApp` 内通过 `useState` 管理 `currentUser`
- 登录：`setCurrentUser(username)` + `sessionStorage.setItem`
- 退出：`setCurrentUser(null)` + `sessionStorage.removeItem` + `setActivePage('home')`

### 路由/导航
- 继续使用现有 SPA 状态路由（`activePage`），不引入 URL 跳转
- `HealthyMealPlannerApp` 最外层先判断 `currentUser`，为 null 则渲染 `LoginPage`，否则渲染完整应用壳

## API 方案
本次不涉及 Worker，不新增 tRPC procedure，不调用 DeepSeek。

## 安全考量
- `sessionStorage` 仅保存用户名，不存密码
- 演示账号明文存在前端源码（本期 MVP 可接受），需在文档和界面说明仅为演示
- 浏览器存储读写包裹 `typeof window !== 'undefined'` 保护，避免 SSR/测试环境异常
- 退出时完整清除 sessionStorage 条目，不遗留残留状态

## 测试策略

### 单元测试
- `src/test/unit/login-page.test.tsx`
  - 空用户名/密码 → 显示"请输入用户名和密码"，`onLogin` 未调用
  - 错误凭证 → 显示"用户名或密码错误"，`onLogin` 未调用
  - 正确凭证 → `onLogin` 以 `{ username: 'demo' }` 被调用

### E2E 测试
- `src/test/e2e/login.spec.ts`
  - 默认进入登录页（不通过 URL 跳转）
  - 错误凭证提示错误
  - 正确登录后进入主应用，导航显示"你好，demo"
  - 刷新后保持登录状态
  - 退出后回到登录页

## 关键技术决策
1. **不引入 Jotai atom**：登录态仅在应用壳层使用，无需跨 feature 共享，用 `useState` 足够且更简洁
2. **sessionStorage 而非 localStorage**：满足"会话内保持登录"需求，关闭标签页自动清除，符合轻量 MVP 安全边界
3. **模拟延迟 400ms**：`window.setTimeout` 模拟网络请求感知，避免登录按钮点击后无反馈，同时保持无真实 API 调用

## 不在设计范围内
- 注册、找回密码、验证码
- 真实后端鉴权 / JWT / Cookie
- 多用户账号管理
- localStorage 持久化登录
