# user-login 需求文档

**版本**：2026-03-29
**状态**：Draft

---

## 背景

FreshPlate 目前是一个开放访问的 SPA，任何人打开即可使用全部功能。随着产品迭代，需要引入最基础的身份认证机制，以便后续支持个人数据持久化（健康计划、收藏菜谱等）。本期目标是快速落地一个轻量登录页，不引入后端鉴权服务，前端硬编码预设账号即可。

---

## 目标

1. 未登录用户打开应用时，默认显示登录页而非直接进入主界面
2. 通过正确的用户名 + 密码登录后进入现有 FreshPlate 主界面，会话刷新后保持
3. 支持从导航栏退出登录，退出后返回登录页

---

## 用户故事

- 作为访客，我希望打开应用时看到登录页，以便保护应用内容不被随意访问
- 作为已注册用户，我希望输入用户名和密码后直接进入首页，以便快速开始使用
- 作为登录用户，我希望刷新页面后不需要重新登录，以便保持流畅体验
- 作为登录用户，我希望点击退出按钮后回到登录页，以便切换账号或结束会话

---

## 功能需求

1. **登录页展示**：应用启动时检测登录状态，未登录则渲染 `LoginPage`，不显示主导航
2. **登录表单**：包含用户名输入框、密码输入框（`type="password"`）、登录按钮
3. **空值校验**：用户名或密码为空时点击登录，阻止提交并在对应输入框下方显示必填错误提示
4. **凭证校验**：前端预设用户名/密码（硬编码），输入不匹配时显示"用户名或密码错误"提示
5. **登录成功**：校验通过后将登录状态写入 `sessionStorage`，切换到主应用首页
6. **会话保持**：页面刷新后读取 `sessionStorage`，已登录则直接进入主界面
7. **退出登录**：顶部导航栏展示当前用户名，提供"退出"按钮；点击后清除 `sessionStorage` 并返回登录页

---

## 验收标准

| # | 验收条件 |
|---|---------|
| AC-1 | 未登录打开应用，默认显示登录页，不显示主导航 |
| AC-2 | 用户名或密码为空点击登录，页面显示必填错误，不跳转 |
| AC-3 | 输入错误用户名或密码，页面显示"用户名或密码错误"，不跳转 |
| AC-4 | 输入正确预设账号后成功进入首页 |
| AC-5 | 登录后刷新页面，保持登录状态，直接显示主界面 |
| AC-6 | 点击退出按钮，返回登录页，sessionStorage 登录状态已清除 |
| AC-7 | 导航栏显示当前登录用户名 |

---

## 非功能需求

- **性能**：登录页首屏渲染 < 1s（无外部依赖）
- **兼容性**：Chrome / Safari / Edge 最新版（与主应用一致）
- **安全**：密码以明文存在前端代码中（本期 MVP 接受），不发起任何网络请求；`sessionStorage` 仅保存用户名，不存密码

---

## 不在范围内

- 注册功能（本期只支持预设账号）
- 忘记密码 / 重置密码
- 第三方登录（Google / GitHub OAuth）
- 后端鉴权服务 / JWT / Cookie
- 多用户账号管理
- 记住密码（localStorage 持久化登录）

---

## 依赖与风险

| 类型 | 描述 |
|------|------|
| 依赖 | Jotai（全局登录状态 atom）、SPA 状态路由（`activePage` 同级添加 `isLoggedIn` 状态） |
| 依赖 | shadcn/ui Input、Button 组件（已有） |
| 风险 | 预设密码硬编码在前端代码中，不适用于生产敏感场景；后续需替换为真实鉴权 |
