# shopping-list-generation 技术设计

**版本**：2026-03-24
**关联需求**：specs/shopping-list-generation/2026-03-24-requirements.md

---

## 方案概述

纯前端方案，无网络请求。在 `HealthPlanPage` 新增弹窗开关 state，将当前 `plan` 数据传给 `ShoppingListModal` 组件；组件内部通过静态映射表 `meal-ingredients-map` 将餐食 title 转换为食材列表，完成去重 + 分类 + 展示。剪贴板写入使用 `navigator.clipboard` API，不支持时隐藏复制按钮。

---

## 影响范围

- **前端**：`src/features/health-plan/`（新增组件、新增 data 文件、扩展 types.ts，修改 health-plan-page.tsx）
- **Worker**：无（无任何 tRPC 调用）
- **共享类型**：无（新类型仅在 health-plan feature 内部使用）

## 数据模型

新增到 `src/features/health-plan/types.ts`：

```typescript
export type IngredientCategory = '蔬果类' | '肉蛋类' | '主食类' | '调料类' | '其他';

export interface IngredientEntry {
  name: string;
  amount: string;        // 参考用量，如 "150g"、"2 个"
  category: IngredientCategory;
}

// 去重合并后的购物清单条目
export interface ShoppingItem {
  name: string;
  category: IngredientCategory;
  amount: string;        // 首次出现的单份用量
  servings: number;      // 累计份数（去重后叠加）
  displayAmount: string; // 展示用量，如 "150g × 3 份"
}
```

## 前端方案

### 新增组件

| 组件 | 路径 | 职责 |
|---|---|---|
| `ShoppingListModal` | `src/features/health-plan/components/shopping-list-modal.tsx` | 弹窗主体：接收 plan，计算购物清单，展示分组列表，提供勾选/复制/重置操作 |

### 新增数据文件

**`src/features/health-plan/data/meal-ingredients-map.ts`**

```typescript
import type { IngredientEntry } from '@/features/health-plan/types';

// key: meal.title（精确匹配，用于覆盖所有 28 条默认餐食）
export const mealIngredientsMap: Record<string, IngredientEntry[]> = {
  '希腊酸奶和浆果': [
    { name: '希腊酸奶', amount: '200g', category: '肉蛋类' },
    { name: '蓝莓', amount: '50g', category: '蔬果类' },
    { name: '草莓', amount: '50g', category: '蔬果类' },
  ],
  '藜麦鸡肉沙拉': [
    { name: '藜麦', amount: '80g', category: '主食类' },
    { name: '鸡胸肉', amount: '150g', category: '肉蛋类' },
    { name: '生菜', amount: '100g', category: '蔬果类' },
    { name: '橄榄油', amount: '10ml', category: '调料类' },
  ],
  // … 覆盖全部 28 条默认餐食（实现阶段补全）
};
```

匹配逻辑（在组件内计算）：
1. 遍历 `plan[].meals[]`，以 `meal.title` 查 map
2. 命中 → 取 `IngredientEntry[]`；未命中 → 生成占位 `{ name: meal.title + ' 所需食材', amount: '适量', category: '其他' }`
3. 按 `name` 去重：相同 name 的 `servings += 1`，`displayAmount` 更新为 `amount + ' × N 份'`
4. 按 `category` 分组，固定顺序：蔬果类 → 肉蛋类 → 主食类 → 调料类 → 其他

### 状态设计

**不新增 Jotai atom**，所有状态用 `useState` 管理：

| 状态 | 位置 | 类型 | 说明 |
|---|---|---|---|
| `isShoppingListOpen` | `HealthPlanPage` | `boolean` | 控制购物清单弹窗开关 |
| `checked` | `ShoppingListModal`（内部） | `Set<string>` | 已勾选的食材 name 集合，弹窗关闭时随组件销毁自动重置 |
| `copyFeedback` | `ShoppingListModal`（内部） | `boolean` | 控制"已复制！"提示的显示/隐藏（1.5s timeout） |

### 路由/导航

无新页面。弹窗通过 `isShoppingListOpen` 控制条件渲染，与 `EditPlanModal` 同级挂载在 `HealthPlanPage` 底部。

空状态"去编辑计划"按钮行为：`onClose()` 关闭购物清单弹窗 + `onOpenEditPlan()` 打开编辑弹窗（回调由 `HealthPlanPage` 注入）。

## 安全考量

- **无 API Key 风险**：纯客户端计算，无网络请求。
- **localStorage 读取**：`HealthPlanPage` 已有完善的读取 + 结构验证 + 容错回退逻辑（`loadFromStorage`），`ShoppingListModal` 直接接收已验证的 `plan` prop，无需再读 localStorage。
- **剪贴板 API**：`navigator.clipboard.writeText` 仅在 HTTPS 上下文可用；渲染时检测 `typeof navigator.clipboard !== 'undefined'`，不可用则隐藏"复制文字"按钮，避免运行时报错。
- **用户输入**：无用户自定义输入写入 prompt 或 DOM（映射表为静态数据），XSS 风险为零。
