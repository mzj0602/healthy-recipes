# shopping-list-generation 需求文档

**版本**：2026-03-24
**状态**：Draft

---

## 背景

FreshPlate 健康计划页（HealthPlanPage）标题区已有"生成购物清单"橙色按钮，但当前点击无任何响应（纯静态占位）。用户完成饮食计划编辑后，自然期望能一键生成本周所需食材采购清单，减少手动整理成本。本功能将打通"编辑计划 → 生成购物清单"的完整闭环。

---

## 目标

1. 用户点击"生成购物清单"后，能立即看到基于当前周计划餐食推导出的食材清单弹窗。
2. 用户可在清单内勾选/取消已购食材，并能导出或复制清单文本。
3. 购物清单与饮食计划数据保持联动——计划更新后重新生成即反映最新数据。

---

## 用户故事

- 作为健康饮食用户，我希望点击"生成购物清单"后弹出本周需要采购的食材列表，以便去超市时不遗漏任何食材。
- 作为忙碌的上班族，我希望能一键复制购物清单文本，以便粘贴到手机备忘录或发给家人。
- 作为已完成部分采购的用户，我希望能勾选已购食材，以便追踪采购进度。

---

## 功能需求

### FR-1 触发入口
- 点击健康计划页标题区"生成购物清单"按钮，弹出 `ShoppingListModal`。
- 弹窗覆盖全屏蒙层，居中展示，可通过 ×、蒙层点击或 Esc 关闭。

### FR-2 食材数据来源
- 从 `localStorage('health-plan-data')` 读取当前周计划的七天餐食列表。
- 每个餐食（`DayPlan.meals`）对应一组预设食材，通过静态映射表（`meal-ingredients-map`）匹配：以 `meal.title` 关键词为键，返回食材数组（名称 + 参考用量）。
- 若餐食名称无法匹配映射表，显示为"通用食材：[meal.title] 所需食材"占位，不报错。
- 去重规则：相同食材名称合并，用量累加并标注单位（如"鸡胸肉 × 3 份"）。

### FR-3 清单展示
- 弹窗标题："本周购物清单（共 N 种食材）"。
- 食材按分类分组展示（蔬果类 / 肉蛋类 / 主食类 / 调料类 / 其他），每组有折叠标题。
- 每条食材：复选框 + 食材名称 + 参考用量，勾选后文字变灰并加删除线。
- 勾选状态存储在组件本地 state（`Set<string>`），弹窗关闭后重置。

### FR-4 操作功能
- **复制清单**：点击"复制文字"按钮，将未勾选食材以纯文本格式（每行一条，格式 `□ 食材名 用量`）写入剪贴板，并显示"已复制！"短暂提示（1.5s 后消失）。
- **重置勾选**：点击"取消全部勾选"恢复所有复选框。

### FR-5 空状态
- 若当前周计划为空（无餐食），弹窗显示空状态插图 + 提示文字"当前计划为空，请先编辑饮食计划"，并提供"去编辑计划"按钮（点击关闭弹窗并触发 EditPlanModal）。

## 验收标准

| # | 场景 | 期望结果 |
|---|------|---------|
| AC-1 | 点击"生成购物清单"按钮 | `ShoppingListModal` 在 300ms 内弹出，标题显示食材总数 |
| AC-2 | 弹窗列表 | 食材按分类分组，去重合并，每条有复选框 |
| AC-3 | 勾选食材 | 文字变灰 + 删除线，再次点击恢复 |
| AC-4 | 点击"复制文字" | 剪贴板内容为未勾选食材的纯文本，页面显示"已复制！"提示 |
| AC-5 | 点击 × / 蒙层 / Esc | 弹窗关闭，勾选状态重置 |
| AC-6 | 当前计划无餐食 | 弹窗显示空状态，"去编辑计划"按钮可用 |
| AC-7 | 编辑计划后重新点击按钮 | 清单反映最新餐食数据 |

---

## 非功能需求

- **性能**：弹窗渲染（含去重计算）< 100ms，食材数据为纯静态计算，无网络请求。
- **兼容性**：Chrome / Safari / Edge 最新版，桌面端 1280px+，剪贴板 API 降级处理（不支持时隐藏复制按钮）。
- **安全**：无 API 调用，无用户数据上传；仅读取 localStorage，不修改计划数据。

---

## 不在范围内

- AI 智能推导食材（不调用 DeepSeek，使用静态映射表）。
- 购物清单持久化到 localStorage 或服务端。
- 打印功能。
- 多人共享购物清单。
- 移动端适配（当前产品仅支持桌面端）。

---

## 依赖与风险

| 项目 | 说明 |
|------|------|
| 依赖 | `localStorage('health-plan-data')` 数据格式须与 `EditPlanModal` 保存格式一致 |
| 依赖 | 需新建 `meal-ingredients-map` 静态数据文件，覆盖现有默认餐食名称 |
| 风险 | 剪贴板 API（`navigator.clipboard.writeText`）在非 HTTPS 环境下不可用，需降级处理 |
| 风险 | 静态映射表初期仅覆盖默认数据中的餐食名称，自定义添加的餐食将走"通用食材"占位 |
