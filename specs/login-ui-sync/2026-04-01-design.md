# login-ui-sync 技术设计

**版本**：2026-04-01
**关联需求**：specs/login-ui-sync/2026-04-01-requirements.md

## 方案概述

对 `login-page.tsx` 进行纯 UI 层改造：将所有硬编码 hex 颜色替换为 `styles.css` 中已定义的 Tailwind token（`bg-primary`、`text-foreground` 等），将原生 `<input>` 和 `<button>` 替换为 shadcn/ui 的 `<Input>` 和 `<Button>` 组件，保留全部业务逻辑不变。改造后登录页颜色、圆角、焦点环与全站设计系统完全一致，未来修改主题只需改 CSS 变量即可全局同步。

## 影响范围

- 前端：`src/features/meal-planner/components/login-page.tsx`（唯一修改文件）
- Worker：无
- 共享类型：无

## 数据模型

无新增类型，仅 UI 改造。`LoginPageProps` 接口和 `demoAccount` 常量保持不变。

#### 2. 用户名 `<input>` → `<Input>`

**原代码**（第 72-80 行）：
```tsx
<input
  id="username"
  type="text"
  autoComplete="username"
  value={username}
  onChange={(event) => setUsername(event.target.value)}
  placeholder="请输入用户名"
  className="w-full rounded-xl border border-[#e7d7ca] bg-[#fffdfb] py-3 pl-10 pr-4 text-sm text-[#3f2617] outline-none transition focus:border-[#ec7f13] focus:ring-4 focus:ring-[#ec7f13]/10"
/>
```

**新代码**：
```tsx
<Input
  id="username"
  type="text"
  autoComplete="username"
  value={username}
  onChange={(event) => setUsername(event.target.value)}
  placeholder="请输入用户名"
  className="py-3 pl-10 pr-4"
/>
```

> 说明：`<Input>` 已内置 `w-full rounded-[14px] border border-border-input bg-white text-sm text-foreground shadow-sm outline-none transition focus-visible:ring-4 focus-visible:ring-ring/30`，仅需传入 `pl-10`（为图标留空间）、`py-3`（保持原高度）、`pr-4`。

#### 3. 密码 `<input>` → `<Input>`

**原代码**（第 90-98 行）同上结构，替换方式一致：
```tsx
<Input
  id="password"
  type="password"
  autoComplete="current-password"
  value={password}
  onChange={(event) => setPassword(event.target.value)}
  placeholder="请输入密码"
  className="py-3 pl-10 pr-4"
/>
```

#### 4. 登录 `<button>` → `<Button>`

**原代码**（第 108-114 行）：
```tsx
<button
  type="submit"
  disabled={isSubmitting}
  className="w-full rounded-xl bg-[#ec7f13] px-4 py-3 text-sm font-semibold text-white transition hover:bg-[#c7670b] disabled:cursor-not-allowed disabled:opacity-70"
>
  {isSubmitting ? '登录中...' : '登录'}
</button>
```

**新代码**：
```tsx
<Button
  type="submit"
  variant="default"
  size="lg"
  disabled={isSubmitting}
  className="w-full font-semibold"
>
  {isSubmitting ? '登录中...' : '登录'}
</Button>
```

> 说明：`<Button variant="default" size="lg">` 已内置 `bg-primary text-primary-foreground h-11 rounded-[16px] px-6`，与原按钮视觉效果一致。hover/disabled 状态由 Button 组件统一管理。仅追加 `w-full font-semibold`。

## 不在设计范围内

- 不改动登录逻辑（校验、会话保存、退出流程）
- 不添加新字段（注册、找回密码、社交登录）
- 不做响应式移动端适配（与全站一致，以桌面端为主）
- 不引入新的 npm 依赖（`<Input>`、`<Button>`、`cn` 均已存在）
- 不修改 `site-chrome.tsx` 或应用入口的登录态管理逻辑
- 不修改 `src/components/ui/input.tsx` 或 `button.tsx` 组件本身

## 关键决策

1. **Input 图标保持 absolute 定位方案**：shadcn/ui `<Input>` 不内置 icon slot，因此保留原有的 `relative` 包裹层 + `absolute` 定位图标方案，通过 `className="py-3 pl-10 pr-4"` 传入左内边距为图标留空间。这比修改 Input 组件本身更安全，避免影响其他使用 Input 的页面。

2. **使用 `size="lg"` 保持按钮高度**：原按钮使用 `py-3` 实际高度约 44px，Button 组件 `size="lg"` 为 `h-11`（44px）+ `rounded-[16px]`，与原视觉效果最接近。选择 `size="lg"` 而非在 `size="default"` 上覆盖高度，遵循组件 API 设计意图。

3. **色值微调为预期行为**：部分硬编码色值（如标题 `#3f2617`、副标题 `#8b6b55`）与对应 token（`--foreground: #1a1a1a`、`--muted-foreground: #64748b`）存在色差。这是设计系统统一的预期结果——登录页应与全站使用同一套色彩规范，而非维护独立的暖棕色调。
